# **CP353: Systems Analysis and Design Homework 1**
---

ในคลิปวีดีโอนี้เป็นการบรรยายของ Martin Fowler ในงาน OOP2014 หัวข้อการบรรยายคือ Workflows of Refactoring โดยเขาเสนอแนวทางในการ Refactor โค้ดในสถานการณ์ต่าง ๆ เริ่มจากสถานการณ์แรกนั่นคือ Cycle of Test Driven Development (TTD) หรือที่คนส่วนมาก
มักจะเรียกกันว่า Red Green Refactor โดยเมื่อเราอยากจะเพิ่ม Function อะไรก็ตามที่เป็น Function ใหม่เข้าไป เราจะการเขียน Test Case เพื่อให้โปรแกรมไม่ผ่านการทดสอบ (Red) เพื่อเป็นจุดเริ่มต้นให้เราเกิดการแก้ไขและทำให้โปรแกรมมันผ่าน Test Case นี้ให้ได้ (Green) เมื่อเรา
มั่นใจแล้วว่าโปรแกรมทำงานถูกต้อง ก็จะเข้าสู่กระบวนการถัดไปนั่นคือ Refactor คือการทำให้โค้ดดู Clean มากยิ่งขึ้น ซึ่งเขาได้มีการเปรียบเทียบกับหมวก 2 ใบ (Two Hats) หากเราจะเขียนโค้ดเราควรจะมี 2 บทบาทที่แยกจากกันโดยชัดเจน จากในวีดีโอเขาได้ยกตัวอย่างว่า หมวกสีเหลือง
คือการเพิ่ม Function ใหม่ ๆ เข้าไป และหมวกสีแดงคือ Refactoring แต่มีสิ่งหนึ่งที่ต้องคำนึงถึงคือการ Refactor ไม่ใช่การทำให้การทำงานของโปรแกรมเปลี่ยนแปลงไปจากเดิม เราต้องรักษาคุณสมบัติเดิมของ Function นั้นเอาไว้
